    使用Github pages + Hexo搭建静态blog

    前言

    一直希望搭建一个blog, 写写文章, 记录下心情和技术, 顺便…体验一下以前那一辈玩转blog的感觉.

    挑了很久, 发现自己选服务器和域名都是一件比较麻烦的事, 恰好听说GitHub pages可以做静态blog, 于是抱着尝试的态度搞了一下.

    教程是使用Hexo框架做的,基于Node.js, 当然, 如果你觉得不喜欢,除了这个还有其他框架, 有些基于python或者ruby的, 网上有很多教程, 大家可以搜搜看.

    ##Github上

    如果没有Github, 请在Github官网上注册一个账号哦. 写到这里我就默认你已经有Github账号了.

    ###创建一个Github仓库

    创建完成后ok, 接下在继续设置一些东西.

    ###将ssh公钥上传

    因为git管理都是通过ssh连接仓库的, hexo的配置文件也是, 所以需要我们配置下ssh.

    1. 首先检查你电脑是否已经有 SSH key

    打开”此电脑 - 系统盘 - 用户(users) - 自己账号文件夹 - .ssh文件夹”, 查看是否存在 id_rsa(私钥)和id_rsa.pub(公钥)文件, 如果有, 则跳过第二步.

    1. 创建一个SSH key

    打开cmd或者Git bash, 输入:

    回车, 提示你输入一个文件名, 推荐使用默认, 然后是输入连接密码, 推荐无, 全部直接按回车, 这样push的时候就不需要输入密码, 直接提交到Github上了.

    1. 添加你的SSH key 到 Github上面去

    首先拷贝 id_rsa.pub 文件的全部内容

    登陆Github, 在右上角的( Account Settings) 进去, 点击菜单栏的SSH key 添加 SSH key. 复制你拷贝的代码到输入框中, 名字随意.
